固有の例外

概要

ほかのライブラリー・コンポーネントによってスローされた例外。

構文

class bad_last_alloc;
class improper_lock;
class invalid_multiple_scheduling;
hclass missing_wait;
class user_abort;

ヘッダー

#include "tbb/tbb_exception.h"

説明

次の表は、例外がスローされるタイミングを示します。

特定の例外用のクラス

例外

スローされるタイミング

bad_last_alloc
  1. concurrent_queue または concurrent_bounded_queue の pop 操作が、例外をスローした push 操作に対応するものであるとき。
  2. 前の操作が例外をスローしたため、concurrent_vector の操作は実行できません。

improper_lock

すでにロックされている critical_section または reader_writer_lock をスレッドがロックしようとしたとき。

invalid_multiple_scheduling

task_group または structured_task_group が task_handle を 2 回実行しようとしたとき。

missing_wait

wait() メソッドが起動される前に task_group または structured_task_group が破棄されたとき。

user_abort

ユーザーによりアボートされた concurrent_bounded_queue の push 操作または pop 操作が呼び出されたとき。

メンバー
